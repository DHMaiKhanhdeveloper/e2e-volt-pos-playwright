## Quan sát thực tế bổ sung từ Playwright (chưa có trong doc phân tích)

- Nút mở form: **"Create Appointment"** (không phải "New Appointment" như doc gốc ghi — cần đối chiếu lại naming, có thể đổi tên qua các version).
- Form Create gồm 3 khối: **Customer Information**, **Service Details**, **Additional Information**.
- Input **Customer Phone** tự động format khi gõ số (vd gõ `19` → hiển thị `(19`).
- **Start time** mặc định là giờ hiện tại làm tròn lên 15 phút.
- **Duration** có 2 combobox cạnh nhau (giờ + phút) — mặc định "1h 0m".
- **Staff** và **Service** là các nút dropdown riêng, hiển thị placeholder "Select staff" / "Select service" khi chưa chọn.
- Nút **"Save Appointment"** ở trạng thái **disabled** ngay khi mở form (chưa nhập gì).
- Checkbox tags **Requested / 🚩 Highlight / No-show** hiển thị dạng 3 checkbox độc lập → có thể multi-select đồng thời (chưa thấy Repeat tag xuất hiện — đúng như doc mô tả "Repeat chỉ hiện khi setting Repeat đang ON").
- Đóng dialog bằng nút "Close" hoạt động, không có cảnh báo "unsaved changes" khi đã gõ dở dữ liệu.
- Data model (comment Thanh Lam 2026-06-22): 1 appointment = 1 parent (`parent_id = NULL`) + N child (line). Create = insert parent+children trong 1 transaction. Edit đổi line = xoá hết child cũ + tạo lại (không update tại chỗ). Edit sửa parent (date/customer/tags/note/status/line đầu) = update tại chỗ theo `id`, không xoá-tạo-lại parent (tránh phá liên kết đồng bộ GoCheckIn).
